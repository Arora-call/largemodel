# 工程项目创建 — 详细更改计划

> 作者：yx | 更新时间：2026-06-18

---

## 一、现状分析

### 当前问题

- 工程模式复用聊天界面，代码/文字/目录树用正则拼凑，AI 格式不可控
- 多文件项目所有代码粘在一起，无法按文件查看、编辑
- 前端项目无法整体预览（只能预览第一个 HTML/Vue 片段）
- "工程项目"和"代码生成"切换概念模糊，用户不知道区别

### 目标

将工程模式从聊天界面中独立出来，做成**项目创建**专用界面：
- 用户输入需求 → AI 生成项目结构 + 多文件代码
- 左侧文件树、中间代码查看、右侧预览（前端项目）
- 点击文件切换代码、前端支持元素级编辑修改

## 三、数据模型设计

### 3.1 后端实体扩展

`Application` 表已有字段足够，`configJson` (JSON 类型) 存储项目结构：

```json
{
  "type": "ENGINEERING",
  "files": [
    { "path": "src/components/DataTable.vue", "language": "vue", "content": "..." },
    { "path": "src/App.vue", "language": "vue", "content": "..." },
    { "path": "package.json", "language": "json", "content": "..." },
    { "path": "vite.config.js", "language": "javascript", "content": "..." }
  ],
  "dependencies": ["vue@3", "element-plus"],
  "entryFile": "src/App.vue",
  "projectType": "frontend"
}
```

### 3.2 AI 输出格式要求

新 Prompt 要求 AI **不输出目录树**，只输出文件列表 + 内容（便于解析）：

```
[PROJECT_TYPE] frontend

[FILE] src/App.vue
```vue
<template>...</template>
```

[FILE] src/main.js
```javascript
import { createApp } from 'vue'
...
```

[FILE] package.json
```json
{ "name": "vue3-table", ... }
```
```

后端新增 `parseProjectResponse(raw)` 方法：
- 提取 `[PROJECT_TYPE]` → frontend / backend
- 按 `[FILE] path` 分割 → 提取每个文件的代码
- 存入 `configJson.files[]`

### 3.3 前端数据流

```
AI 响应 → 后端解析 → configJson.files[] → 前端 files[] 响应式数组
                                              │
                          ┌───────────────────┼───────────────────┐
                          ▼                   ▼                   ▼
                      文件树组件          代码查看器           预览 iframe
                   (递归渲染 files)   (Monaco Editor)    (前端项目整体)
```

## 四、组件设计

### 4.1 新增文件

```
src/
├── views/
│   └── project/
│       └── Create.vue          ← 项目创建主页面
├── components/
│   ├── project/
│   │   ├── FileTree.vue        ← 文件树组件
│   │   ├── FileCode.vue        ← 文件代码查看/编辑
│   │   └── ProjectPreview.vue  ← 前端项目预览面板
│   └── CodeViewer.vue          ← 已存在（复用）
├── api/
│   └── project.js              ← 工程 API
└── router/
    └── index.js                ← 新增路由 /project/create
```

### 4.2 FileTree.vue

```
Props:
  files: Array<{ path, language, content }>
  activeFile: String

Emits:
  select: (filePath)

功能：
  - 将 files[] 按路径拆分为树形结构
  - 文件夹可折叠/展开
  - 点击文件高亮并 emit select
  - 文件图标按语言区分（Vue→🟢, Java→🟠, JSON→🔵）
```

### 4.3 FileCode.vue

```
Props:
  file: { path, language, content } | null
  editable: Boolean

Emits:
  update: (newContent)

功能：
  - 复用 CodeViewer（Monaco Editor）
  - 编辑模式下可修改代码
  - 显示文件路径 + 语言标签
  - 复制、保存按钮
```

### 4.4 ProjectPreview.vue

```
Props:
  files: Array<{ path, content }>
  projectType: 'frontend' | 'backend'

功能：
  - 仅 frontend 项目显示
  - 将 Vue/HTML 项目打包为可预览的 HTML
  - iframe 沙箱渲染
  - 编辑模式下拾取元素（复用现有 buildPreviewHtml + postMessage）
```

## 五、后端改动

### 5.1 新增 API

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/projects/generate` | 流式生成工程项目（SSE） |
| `POST` | `/api/projects/{id}/modify` | 修改单个文件 |
| `GET` | `/api/projects/{id}` | 获取项目详情（含 files） |
| `PUT` | `/api/projects/{id}/files/{path}` | 更新单个文件内容 |
| `GET` | `/api/projects/{id}/preview` | 生成预览 HTML（前端项目） |

### 5.2 AI 服务改动

`AiCodeGenService` 新增：
- `generateProject(request, userId)` — 专用工程 SSE 流式方法
- `parseProjectResponse(raw)` — 解析 `[PROJECT_TYPE]` + `[FILE]` 标记
- `modifyFile(projectId, filePath, prompt)` — 单文件增量修改

### 5.3 Prompt 改动

`PromptTemplateService` 新增：
- `getProjectSystemPrompt()` — 新 Prompt，要求 AI 按 `[FILE]` 标记输出

## 七、交互流程

```
1. 用户进入 /project/create
2. 看到输入框 + 示例提示词
3. 输入「创建 Vue3 数据表格组件，支持分页和排序」
4. 点击生成 → SSE 流式展示进度
5. 生成完成：
   ├── 左侧展示文件树（src/App.vue, src/components/DataTable.vue, ...）
   ├── 默认选中第一个文件，中间显示代码
   └── 右侧预览面板渲染整体效果（前端项目）
6. 点击文件树中的文件 → 中间切换代码
7. 预览面板切换「编辑」模式 → hover 高亮元素 → 点击选中
8. 底部输入修改需求 → AI 增量修改该文件
9. 点击保存 → 存入 applications 表
10. 可在「我的应用」中查看/下载已保存的项目
```

## 八、与现有功能的兼容

| 现有功能 | 处理方式 |
|---------|---------|
| 聊天式代码生成（NATIVE） | **保留不变**，路径 `/ai/generate` |
| 工程模式切换 | **移除**，聊天页面不再有工程/代码切换 |
| 对话历史 | 保留，仅影响代码生成模式 |
| 我的应用（AppList） | 保留，支持展示两种类型的项目 |
| 下载 | 保留，前端项目 ZIP 打包、后端项目同理 |
